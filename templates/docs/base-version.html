Base version

When you first run the siphon create command, we automatically assign your app a special value called a base_version behind the scenes. Open up the file named Siphonfile in your app's directory to see the currently assigned version.

It ties your app to a particular version of React Native and also guarantees that a certain set of native modules are available (e.g. react-native-video) to be used within your app's JavaScript code. You don't need to link these native libraries yourself.

Every few weeks we add a new base version to the list, which includes new native modules and usually the latest version of React Native.

Changing the base version of an app

The base_version assigned to an app is controlled by the file named Siphonfile that appears in every Siphon app directory. It uses the JSON format and looks something like this:

{% include "docs/snippets/base-version.html" %}

To change the base version, edit the Siphonfile and run a siphon push to reflect the change.

If your app is already published to the App Store or Play Store, you will need to run the siphon publish command again if you would like to change the base version of your production app.
